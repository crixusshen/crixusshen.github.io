数据验证

数据验证

主要用于验证Controller所接受的请求参数(即请求URL的参数数据验证)，一旦验证失败，会抛出一个 status = 422 的异常，你可以通过统一异常来进行异常归集和处理

下面展示了一个最简单的数据验证实现，通过定义规则从而实现数据规则的验证，一般开发者需要对请求URL的传入参数做好必要的校验，通过后方可调用Service进行逻辑处理： 

// src/{application}/app/controller/exceptionHandler.js:
const Controller = require('fastman2-middleground-framework').Controller;

// 定义创建接口的请求参数规则
const createRule = {
  accessToken: 'string',
  title: 'string',
  tab: {
    type: 'enum',
    values: [ 'ask', 'share', 'job' ],
    required: false,
  },
  content: 'string',
};

class exceptionHandlerController extends Controller {
  async create() {
    const ctx = this.ctx;
    // 校验 `ctx.request.body` 是否符合我们预期的格式
    // 如果参数校验未通过，将会抛出一个 status = 422 的异常
    ctx.validate(createRule, ctx.request.body);

    // 调用Service前 ...
  }
}

module.exports = exceptionHandlerController;

规则定义

每条规则的定义名称即是this.ctx.request.body中的请求参数名，可以通过对象和字符串来定义规则，使用字符串定义时即是type的定义，具体如下： 

const createRule = {
  // 对象形式定义
  param1: {
    // 属性的类型，每种类型都有自己的验证规则
    type: 'string',
    // 使参数将输入参数转换为特定类型，支持int、number、string和boolean，还支持自定义自己的转换方法的函数(即将参数强制转化为该定义的类型)
    convertType: 'string',
    // 如果设置为false，则参数允许null或undefined，默认值为true
    required: true,
    // 默认值，当未传入所定义的参数或者参数不允许空(即required=false)，该参数会被赋值为该默认值，一般我们可以使用这个定义来实现一些参数的默认值定义
    default: 'xxx',
  },
  // 字符串形式定义
  param2: 'int',
}

添加额外规则

如果规则无法满足需求，开发者可添加额外的规则来满足需求，具体实现如下： 

app.validator.addRule('jsonString', (rule, value) => {
  try {
    JSON.parse(value);
  } catch (err) {
    return 'must be json string';
  }
});
int

如果type为int，有以下二个额外的规则可定义：

  • max - 该参数的最大值，值必须<=max
  • min - 该参数的最小值，值必须>=min

默认的convertType为int。

number

如果type为number，有以下二个额外的规则可定义：

  • max - 该参数的最大值，值必须<=max
  • min - 该参数的最小值，值必须>=min

默认的convertType为number。

date

如果type为date，值会匹配类似YYYY-MM-DD的字符串格式。

默认的convertType为string。

dateTime（也可以定义为datetime）

如果type为dateTime或datetime，值会匹配类似YYYY-MM-DD HH:mm:ss的字符串格式。

默认的convertType为string。

id

如果type为id，值会匹配/^\d+$/正则表达式的日期字符串，类似052111。

默认的convertType为string。

boolean（也可以定义为bool）

匹配布尔类型。

默认的convertType为boolean。

string

如果type为string，有以下四个额外的规则可定义：

  • format - 一个正则表达式来验证字符串值
  • max - 字符串的最大长度
  • min - 字符串的最小长度
  • trim - 在校验前对字符串进行空格过滤，默认该值为false

默认的convertType为string。

email

匹配RFC 5322规范的邮箱格式。

默认的convertType为string。

url

匹配web url格式的字符串。

默认的convertType为string。

enum

如果type为enum，即枚举类型，有一个额外的规则可定义：

  • values - 一个数组，参数必须是该数组中的某个值，该规则为必填项。
object

如果type为object，即对象类型，有一个额外的规则可定义：

  • rule - 可定义对象内属性的规则。
array

如果type为array，即数组类型，有四个额外的规则可定义：

  • itemType - 数组成员的类型定义
  • rule - 如果itemType为object，则该字段可定义对象内的属性规则
  • max - 数组的最大成员数量
  • min - 数组的最小成员数量
