消息结构

消息结构

小服务可在框架中灵活的定义符合业务自身的消息结构体。

在helper中定义消息成功结构： 

// src/app/extend/helper.js:  
module.exports = {
  success(data = null, info = 'ok', status = 200) {
    this.ctx.body = {
      // 消息头 - 主要存放一些公共数据，可扩展
      header: {
        // 当前用户登录态（true:已登录；false:未登录或登录态已失效）
        state: true,
        // 服务端返回的时间戳，客户端可通过该字段换算当前服务端时间，在某些特殊业务流程中可能需要服务端的时间
        timestamp: new Date().getTime(),
      },
      // 消息载体 - 存放业务调用成功和失败错误码、调用描述以及业务数据
      payload: {
        // 成功码，默认0
        code: 0,
        // 成功描述，默认ok
        info,
        // 业务数据，是一个对象
        data,
      },
    };
    this.ctx.status = status;
  },
};

在helper中定义消息失败结构，也可以重写this.app.BizErrorHandler来实现对业务错误的重写： 

// app.js
async didLoad() {
    // 重写默认的业务异常处理器，一般可生成自定义的消息结构
    this.app.BizErrorHandler = class extends this.app.BizErrorHandler {
      json(ctx, error, config) {
        const body = {
          header: {
            state: true,
            timestamp: new Date().getTime(),
          },
          payload: {
            // 错误码，会读取config/errorCode.js中的定义
            code: config.code,
            // 错误描述，会读取config/errorCode.js中的定义
            info: config.message,
          },
        };
        // 输出一些业务字段和校验字段，方便排查错误
        if (config.errors || error.errors) {
          body.payload.detail = Object.assign({}, config.errors, error.errors);
        }
        ctx.body = body;
      }
    };
  }

this.ctx.helper.success(data, [info = 'ok', status = 200])

业务成功调用，其中data则对应成功消息结构中的payload data，而info对应payload info，status对应httpStatus，后二者如果没有定义会采用默认值。

this.ctx.helper.error(code, [bizParams = {}])

业务失败调用，其中code对应config/errorCode.js中定义的错误码对象key，该错误码对象的value中message和code分别会对应失败消息体结构中的payload info和payload code，而bizParams是可选的，你可以将一些非敏感的业务字段传入则会在失败消息体的payload errors中输出，方便进行问题排查，而在生产环境下无论你是否定义bizParams，都不会输出payload errors。
