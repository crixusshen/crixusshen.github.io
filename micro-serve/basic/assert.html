断言库

断言

是一种用于测试约定的辅助工具。目前的主流断言库包含expect、should和assert，其中assert是作为NodeJs官方所推荐的断言库。框架内部则集成了一个更加灵活的断言库，可以极大的避免学习断言语法的学习成本，因此你只需要掌握下面几个API语法即可。同时它具有强大的错误信息反馈能力。 

    assert(types[index].name === bob.name)
           |    ||      |    |   |   |
           |    ||      |    |   |   "bob"
           |    ||      |    |   Person{name:"bob",age:5}
           |    ||      |    false
           |    |11     "alice"
           |    Person{name:"alice",age:3}
           ["string",98.6,true,false,null,undefined,#Array#,#Object#,NaN,Infinity,/^not/,#Person#]

    --- [string] bob.name
    +++ [string] types[index].name
    @@ -1,3 +1,5 @@
    -bob
    +alice

assert(value[, message])

等价于value==true

  • value 判断value（可以是bool值也可以是一个表达式）是否是true
  • value |

assert.ok(value[, message])

与assert(value[, message])等价，是它的别名。

assert.equal(actual, expected[, message])

等价于actual==expected

  • actual 实际值
  • expected 预期值
  • message |

assert.notEqual(actual, expected[, message])

等价于actual!=expected

assert.strictEqual(actual, expected[, message])

等价于actual===expected，即值类型比较。

  • actual 实际值
  • expected 预期值
  • message |

assert.notStrictEqual(actual, expected[, message])

等价于actual!==expected

assert.deepEqual(actual, expected[, message])

如果是引用类型，则比较结构，但是对应的Prototype会被忽略比较。

assert.notDeepEqual(actual, expected[, message])

与assert.deepEqual(actual, expected[, message])保持相反
