接口规范

RESTful

RESTful接口规范可提供一组标准的设计原则和约束条件，基于这个风格设计可以使接口设计变得更加简洁，更有层次，这种风格的API设计特别适合用于后台管理系统的接口设计

使用该规范只需要定义好具体的URL和控制器入口的映射关系，而控制器中的函数定义都已经约定俗成。下面是具体的开发规范：

路由定义

module.exports = app => {
  router.resources(['router-name'], 'path-match', [middleware1, ..., middlewareN], app.controller.fileName);

  // 例如：
  router.resources('restfuls', '/api/restful', controller.restful);
};

和普通路由定义不同的时，这里使用resources来定义接口路由，app.controller.fileName已经不是对应functionName，而是fileName，你只需要在fileName对应的控制器文件中实现对应的函数即可：

Method Path Route Name Controller.Action
GET /api/restful restfuls app.controllers.restful.index
GET /api/restful/new new_restful app.controllers.restful.new
GET /api/restful/:id restful app.controllers.restful.show
GET /api/restful/:id/edit edit_restful app.controllers.restful.edit
POST /api/restful restfuls app.controllers.restful.create
PUT /api/restful/:id restful app.controllers.restful.update
DELETE /api/restful/:id restful app.controllers.restful.destroy

下面是所对应的控制器具体实现，可以看到Controller.Action的命名都是固定和规范的： 

// app/controller/restful.js:
const Controller = require('fastman2-middleground-framework').Controller;

class RESTfulController extends Controller {
  // GET /api/restful
  async index() {
    this.success('GET:index');
  }

  // GET /api/restful/new
  async new() {
    this.success('GET:new');
  }

  // GET /api/restful/:id
  async show() {
    const id = this.ctx.params.id;
    this.success(`GET:show:${id}`);
  }

  // GET /api/restful/:id/edit
  async edit() {
    const id = this.ctx.params.id;
    this.success(`GET:edit:${id}`);
  }

  // POST /api/restful
  // 需要防范CSRF缺陷
  async create() {
    this.success('POST:create');
  }

  // PUT /api/restful/:id
  async update() {
    const id = this.ctx.params.id;
    this.success(`PUT:update:${id}`);
  }

  // DELETE /api/restful/:id
  async destroy() {
    const id = this.ctx.params.id;
    this.success(`DELETE:destroy:${id}`);
  }
}

module.exports = RESTfulController;
